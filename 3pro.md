# Инструкция для работы (Anti-Freeze Mode)

1. **Лимит строк:** Не генерировать ответы длиннее 50-60 строк кода за один раз.
2. **Инструменты:** Всегда использовать `replace_string_in_file` для точечных правок. Никогда не переписывать файл целиком, если меняется только часть.
3. **Разбиение:** Если задача требует много кода, разбивать её на 2-3 последовательных шага (tool calls).
4. **Краткость:** Писать только необходимый код. Использовать комментарии `// ...existing code...` для пропуска неизменяемых частей.
5. **Диалог:** Отвечать пользователю коротко и по делу.
